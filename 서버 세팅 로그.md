# WEB Server

### Apache, SSL

패키지 목록 갱신

```
sudo dnf makecache
```

Apache(httpd) 설치

```
sudo dnf install -y httpd
sudo systemctl enable --now httpd
```

mod_ssl 설치

```
sudo dnf install -y mod_ssl
```

SSL 인증서 업로드 후 권한 세팅

```
sudo chmod 644 /etc/pki/tls/certs/dlanding_dongasocio_com_cert.pem
sudo chmod 644 /etc/pki/tls/private/dlanding_dongasocio_com_key.pem
sudo chmod 644 /etc/pki/tls/certs/Chain_RootCA_Bundle.crt
sudo chown root:root /etc/pki/tls/certs/dlanding_dongasocio_com_cert.pem
sudo chown root:root /etc/pki/tls/private/dlanding_dongasocio_com_key.pem
sudo chown root:root /etc/pki/tls/certs/Chain_RootCA_Bundle.crt
```

- 인증서 비밀번호: DA@112Web

Apache SSL 설정 파일 업로드 후 권한 세팅

```
sudo chown root:root /etc/httpd/conf.d/ssl.conf
sudo chown root:root /etc/httpd/conf.d/rocky-snipolicy.conf
```

80 → 443 리다이렉트 설정

```
cat > /etc/httpd/conf.d/redirect.conf <<'EOF'
<VirtualHost *:80>
    ServerName dlanding.dongasocio.com
    Redirect permanent / https://dlanding.dongasocio.com/
</VirtualHost>
EOF
```

Apache 설정 문법 검사

```
sudo apachectl configtest
```

Apache 재시작

```
sudo systemctl restart httpd
```

Apache 상태 확인

```
sudo systemctl status httpd
```

# WAS Server

### Tomcat:8080

시스템 업데이트

```
sudo dnf -y upgrade
```

Java 21 설치

```
sudo dnf -y install java-21-openjdk
```

Tomcat 설치 파일 업로드 후 기본 경로 세팅

```
cd /tmp
sudo mkdir -p /opt/tomcat-8080
sudo tar -xzf apache-tomcat-11.0.18.tar.gz -C /opt/tomcat-8080 --strip-components=1
sudo chown -R dainfo:dainfo /opt/tomcat-8080
sudo chmod +x /opt/tomcat-8080/bin/*.sh
```

`/opt/tomcat-8080/conf/server.xml` 파일에 내용 추가

```
<Valve className="org.apache.catalina.valves.RemoteIpValve"
       remoteIpHeader="X-Forwarded-For"
       protocolHeader="X-Forwarded-Proto"
       protocolHeaderHttpsValue="https"
       portHeader="X-Forwarded-Port"
       requestAttributesEnabled="true" />
```

실행 확인

```
/opt/tomcat-8080/bin/catalina.sh start
/opt/tomcat-8080/bin/catalina.sh stop
```

### Git + Apache

Apache 설치

```
sudo dnf install -y httpd httpd-tools git
sudo systemctl enable --now httpd
sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --reload
```

최초 인증용 사용자 만들기

```
sudo htpasswd -c /etc/httpd/conf/.gitpasswd a2023200
```

- 사용자 추가 방법: `sudo htpasswd /etc/httpd/conf/.gitpasswd a2023200`
- 사용자 삭제 방법: `sudo htpasswd -D /etc/httpd/conf/.gitpasswd a2023200`
- 사용자 목록 확인: `sudo cat /etc/httpd/conf/.gitpasswd`

Apache + Git 설정 파일 생성

```
sudo tee /etc/httpd/conf.d/git.conf > /dev/null <<'EOF'
SetEnv GIT_PROJECT_ROOT /srv/git
SetEnv GIT_HTTP_EXPORT_ALL
ScriptAlias /git/ /usr/libexec/git-core/git-http-backend/

<Location /git>
AuthType Basic
AuthName "Git"
AuthUserFile /etc/httpd/conf/.gitpasswd
Require valid-user
</Location>
EOF
```

`receivepack true` 설정

```
sudo -u git git --git-dir=/srv/git/lpbe.git config http.receivepack true
```

Apache 프로세스가 저장소를 읽고 쓸 수 있도록 설정

```
sudo chgrp -R apache /srv/git/lpbe.git
sudo chmod -R g+rwX /srv/git/lpbe.git
sudo find /srv/git/lpbe.git -type d -exec chmod g+s {} \;
```

시스템 전역으로 `safe.directory` 등록

```
sudo git config --system --add safe.directory /srv/git/lpbe.git
```

확인

```
sudo git config --system --get-all safe.directory
```

Apache 설정 문법 검사

```
sudo apachectl configtest
```

Apache 재시작

```
sudo systemctl restart httpd
```

Apache 상태 확인

```
sudo systemctl status httpd
```

### Jenkins

시스템 업데이트

```
sudo dnf -y upgrade
```

Java 21 설치

```
sudo dnf -y install java-21-openjdk
sudo dnf -y install java-21-openjdk-devel
```

nodejs 설치

```
sudo dnf module reset -y nodejs
sudo dnf module enable -y nodejs:24
sudo dnf install -y nodejs npm
```

Jenkins LTS 저장소 키 등록

```
sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io-2023.key
```

Jenkins 저장소 추가

```
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/rpm-stable/jenkins.repo
```

Jenkins 설치

```
sudo dnf -y install jenkins
```

서비스 등록 및 시작

```
sudo systemctl enable --now jenkins
```

상태 확인

```
sudo systemctl status jenkins
```

firewalld 8080/tcp 오픈

```
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

확인

```
sudo firewall-cmd --list-ports
```
