# 개발 환경 세팅 가이드

<aside>

</aside>

이 문서에서는 랜딩 페이지 Backend 개발을 시작하기 위해 필요한 기본 개발 환경 세팅 방법을 안내합니다.

진행 순서는 다음과 같습니다.

1. Windows PowerShell에서 Git 설치 및 세팅
2. IntelliJ IDEA 설치
3. Git 프로젝트 Clone

# 1. Windows PowerShell에서 Git 설치 및 세팅하는 방법

## 1-1. PowerShell 실행

Windows 검색창에서 `PowerShell`을 검색한 뒤, **Windows PowerShell**을 실행합니다.

가능하면 **관리자 권한으로 실행**하는 것을 권장합니다.

## 1-2. Git 설치

PowerShell에서 아래 명령어를 입력합니다.

```powershell
winget install --id Git.Git -e --source winget
```

`winget`은 Windows에서 프로그램을 검색, 설치, 업데이트할 수 있는 명령줄 도구이며, Git 공식 설치 페이지에서도 Windows PowerShell 또는 명령 프롬프트에서 `winget`을 이용한 설치 방법을 안내하고 있습니다.

설치가 완료되면 PowerShell을 종료한 뒤 다시 실행합니다.

## 1-3. Git 설치 확인

아래 명령어를 입력해 Git이 정상적으로 설치되었는지 확인합니다.

```powershell
git --version
```

Git 버전 정보가 출력되면 정상적으로 설치된 것입니다.

예시:

```powershell
git version 2.xx.x
```

## 1-4. Git 사용자 정보 설정

Git 커밋 기록에 사용할 이름과 이메일을 설정합니다.

```powershell
git config --global user.name "이름"
git config --global user.email "이메일주소"
```

예시:

```powershell
git config --global user.name "홍길동"
git config --global user.email "hong@example.com"
```

설정이 정상적으로 적용되었는지 확인하려면 아래 명령어를 입력합니다.

```powershell
git config --global --list
```

아래와 같이 `user.name`, `user.email` 값이 출력되면 정상적으로 설정된 것입니다.

```powershell
user.name=홍길동
user.email=hong@example.com
```

# 2. IntelliJ IDEA 설치하는 방법

## 2-1. IntelliJ IDEA 다운로드 페이지 접속

JetBrains 공식 IntelliJ IDEA 다운로드 페이지에 접속합니다.

IntelliJ IDEA는 Windows, macOS, Linux를 지원하며, JetBrains 공식 다운로드 페이지에서 최신 버전을 받을 수 있습니다.

## 2-2. 버전 선택

IntelliJ IDEA는 일반적으로 아래 두 가지 버전으로 제공됩니다.

- **Ultimate**: Spring Boot 개발에 필요한 기능을 더 많이 지원하는 유료 버전
- **Community Edition**: 무료 버전

무료 버전을 사용해도 개발 진행에 무리가 없습니다.

## 2-3. 설치 파일 실행

다운로드한 설치 파일을 실행한 뒤 안내에 따라 설치를 진행합니다.

설치 중 별도 설정이 필요하지 않다면 기본 옵션으로 진행해도 괜찮습니다.

설치가 완료되면 IntelliJ IDEA를 실행합니다.

## 2-4. 초기 설정

IntelliJ IDEA를 처음 실행하면 JetBrains 계정 로그인, 테마, 플러그인 설정 등을 진행할 수 있습니다.

특별히 필요한 설정이 없다면 기본값으로 진행해도 괜찮습니다.

# 3. Git 프로젝트 Clone 하는 방법

## 3-1. 프로젝트를 저장할 폴더로 이동

PowerShell에서 프로젝트를 저장할 위치로 이동합니다.

예를 들어 `C:\\workspace` 폴더에 프로젝트를 저장하려면 아래와 같이 입력합니다.

```powershell
cd C:\\workspace
```

폴더가 없다면 먼저 생성합니다.

```powershell
mkdir C:\\workspace
cd C:\\workspace
```

## 3-2. Git 저장소 URL 복사

- http://172.16.4.247/git/lpbe.git
- http://172.16.4.247/git/lpbe_scheduler.git

## 3-3. 프로젝트 Clone

PowerShell에서 아래 명령어를 입력합니다.

```powershell
git clone <저장소 URL>
```

`git clone` 명령어는 원격 저장소를 로컬 PC의 새 디렉터리로 복사하는 명령어입니다. Git 공식 문서와 GitHub 문서에서도 저장소 URL을 복사한 뒤 `git clone <URL>` 형식으로 실행하는 방식을 안내하고 있습니다.

Clone이 완료되면 아래와 같이 프로젝트 폴더로 이동합니다.

```powershell
cd landing-backend
```

## 3-4. IntelliJ IDEA에서 프로젝트 열기

IntelliJ IDEA를 실행한 뒤 아래 순서로 프로젝트를 엽니다.

```
File > Open
```

이후 Clone 받은 프로젝트 폴더를 선택합니다.

프로젝트를 처음 열면 Gradle 설정을 불러오는 시간이 걸릴 수 있습니다. 오른쪽 하단 또는 상단에 표시되는 안내가 있다면 **Load Gradle Project**, **Trust Project** 등을 선택해 프로젝트 설정을 완료합니다.
